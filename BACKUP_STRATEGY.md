为防止关键依赖项目突然下架或停止维护，File Brain 建立了本地备份机制。

## 备份方案

### 1. 本地备份
```bash
# 备份所有依赖源码
pip download -r requirements.txt -d backup_deps/

# 定期更新备份
# 每月执行一次
```

### 2. 私有仓库镜像
- 在 GitHub 上创建私有仓库
- 镜像关键依赖项目
- 定期同步更新

### 3. 替代方案准备
- 为每个关键依赖准备替代方案
- 定期评估替代方案的可行性
- 保持技术栈的灵活性

## 应急计划

### 依赖下架应急流程
1. **立即响应**：切换到本地备份版本
2. **短期方案**：使用替代依赖
3. **长期方案**：fork 并自行维护

### 关键依赖替代方案

#### sentence-transformers → 替代方案
- 使用 Hugging Face Transformers 直接实现
- 使用 ONNX 模型本地推理
- 使用 OpenAI API（需付费）

#### scikit-learn → 替代方案
- 使用 numpy + scipy 实现核心算法
- 使用纯 Python 实现 TF-IDF
- 使用其他机器学习库

#### networkx → 替代方案
- 使用 igraph
- 使用 graph-tool
- 自行实现基础图算法

## 定期审查

- **每月**：检查依赖更新状态
- **每季度**：评估依赖风险
- **每年**：更新备份和替代方案
