# File Brain 项目治理模型

## 概述

File Brain 采用 **BDFL（仁慈的独裁者）+ 维护小组** 的治理模型，确保项目的长期健康发展。

## 角色定义

### 1. BDFL（仁慈的独裁者）

**当前**：Mby159

**职责**：
- 项目的最终决策权
- 设定项目愿景和方向
- 任命维护小组成员
- 处理社区冲突

**限制**：
- 重大变更需征求社区意见
- 尊重维护小组的技术决策
- 透明化决策过程

### 2. 维护小组（Maintainers）

**组成**：3-5 名活跃贡献者

**职责**：
- 代码审查和合并
- Issue 管理和分类
- 发布版本
- 技术指导

**产生方式**：
- 由 BDFL 任命
- 需有显著贡献记录
- 承诺长期参与

### 3. 贡献者（Contributors）

**任何提交 PR 并被合并的人**

**权益**：
- 在 README 中列出
- 参与技术讨论
- 申请成为维护者

## 决策流程

### 日常决策
- 维护者可以直接决策
- 代码审查通过即可合并
- 文档更新无需审批

### 重大决策
- 架构变更
- API 破坏性修改
- 许可证变更
- 需要 BDFL 批准

### 社区投票
- 新功能优先级
- 版本发布计划
- 维护者任命

## 冲突解决

1. **技术争议**：维护小组讨论决定
2. **社区冲突**：BDFL 调解
3. **严重分歧**：社区公开讨论

## 退出机制

### 维护者退出
- 主动辞职
- 长期不活跃（6个月）
- 违反行为准则

### BDFL 交接
- 指定继任者
- 社区认可
- 逐步过渡

---

*本治理模型会根据项目发展适时调整*
